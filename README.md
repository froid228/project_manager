# Серверная часть веб-приложения для управления проектами

Курсовой проект: **серверная часть** веб-приложения для совместного
управления проектами, распределения задач и координации работы команды.
Реализована на **Django** по паттерну **MVT** (Model --- View ---
Template) с параллельным предоставлением **REST API**.

**Нормативная база курсового проектирования:** инструкция по организации
и проведению курсового проектирования СМКО МИРЭА **7.5.1/04.И.05-18**.

7.32-2017**.

## Предметная область (кратко)

Пользователи работают над **проектами**, внутри которых создаются
**задачи**. Каждый проект имеет владельца и список участников с
назначенными ролями. Задачи характеризуются статусом выполнения
(`к выполнению`, `в работе`, `готово`), приоритетом (`низкий`,
`средний`, `высокий`), исполнителем и дедлайном.

Доступ к данным строго регламентирован: **администратор** видит всю
систему, **менеджер** управляет своими проектами, **участник** работает
только с задачами в своих проектах, **наблюдатель** имеет права только
на чтение. Система поддерживает как веб-интерфейс для конечных
пользователей, так и программный API для автоматизации.

## Основной функционал

-   Регистрация и вход, получение JWT-токенов (`/api/token/`)
-   CRUD проектов и задач через веб-интерфейс и REST API
-   Назначение участников в проекты с указанием роли
-   Отслеживание статусов, приоритетов и дедлайнов задач
-   Фильтрация и пагинация списков на уровне API
-   Группы прав доступа на уровне представлений (`permissions.py`)
-   Адаптивный интерфейс с анимациями и уведомлениями
-   Команда `python manage.py seed` для быстрого развёртывания
    демо-данных

## Установка и запуск

### 1. Клонирование и переход в каталог

``` bash
git clone <URL репозитория>
cd project_manager
```

### 2. Виртуальное окружение

``` bash
# Windows
python -m venv venv
venv\Scripts\activate

# macOS / Linux
python3 -m venv venv
source venv/bin/activate
```

### 3. Зависимости

``` bash
pip install -r requirements.txt
```
### 4. Миграции

``` bash
python manage.py makemigrations users projects tasks
python manage.py migrate
```

### 5. Суперпользователь (админ-панель и тестовый вход)

``` bash
python manage.py createsuperuser
```
### 6. Запуск сервера разработки

``` bash
python manage.py runserver
```

Приложение: <http://127.0.0.1:8000/>.

## Модели данных (логический уровень)

-   **User** --- кастомная модель пользователя; поле `role` определяет
    уровень доступа (`admin`, `manager`, `member`, `observer`).
-   **Project** --- название, описание, владелец (`ForeignKey` к User),
    дата создания.
-   **ProjectMember** --- промежуточная таблица связи M:N между User и
    Project; хранит роль участника в конкретном проекте.
-   **Task** --- заголовок, описание, статус, приоритет, исполнитель
    (`ForeignKey`), дедлайн, привязка к проекту (`ForeignKey`),
    временные метки создания/обновления.

Связи и ограничения целостности задаются в моделях и миграциях Django
ORM. Индексы добавлены на часто используемые поля поиска.

## Паттерн MVT в проекте

-   **Model** --- описание сущностей, связей и миграций в
    `users/models.py`, `projects/models.py`, `tasks/models.py`.
-   **View** --- обработка HTTP-запросов, выборка из БД, проверка прав,
    вызов форм и `render()` / возврат `Response` в `core/views.py`
    (веб-сценарии) и `*/views.py` (API-сценарии).
-   **Template** --- HTML-разметка с тегами Django и контекстными
    переменными в `templates/`. Наследование от `base.html` обеспечивает
    единый стиль и навигацию.

Маршрутизация: `config/urls.py` подключает API-маршруты
(`path('api/', include(...))`) и веб-маршруты
(`path('', include('core.web_urls'))`). DRF Router автоматически
генерирует стандартные CRUD-endpoints для ViewSet'ов.
